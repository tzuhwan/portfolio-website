Building An ETL Pipeline for Building Thermal Data

For the energy engineers at the Brown Office of Sustainability, a key challenge is being able to analyze and act on quality campus building utility data as real-time as possible. As an OS data intern in the summer of 2021, I built a data pipeline that automatically extracts, transforms and loads heat-utility data from an energy information database to a Postgresql database, from which we can create meaningful analytics. The raw data originates from 44 thermal sub-meters in different campus buildings that emit utility readings at every 15-min interval. This project is useful because:

  1. In the event of a sub-meter defect detected from the processed data, the energy engineers can respond immediately;
  2. The Office of Sustainability can make Year-Over-Year comparison on different levels of aggregation(quarterly, weekly, daily, hourly) to set specific building utility optimization goals;
  3. With some modifications the scripts can be used for other categories of meters in the energy information database.

I wrote the scripts in R, and created interactive dashboard on Tableau. One of the scripts also automatically generates a report on the weekly usage of the targeted thermal meters and uploads it to a shared Google Drive folder for quick overview acess.
